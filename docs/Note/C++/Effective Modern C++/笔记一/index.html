Effective Modern C++ | 笔记一

模板推导​

模板推导中，我们定义了形参类型（ ParamType ）与类型（ T ），这两者并不相同

考虑下面这个模板

template<typename T>
void f(const T& param);         //ParamType是const T&

如果我们传入以下实参

int x = 0;
f(x);                           //T被推导为int, ParamType被推导为const int&

可以看到 ParamType 和T 被推导为了不同的类型

在实际情况中， T 的推导取决于 expr 的类型，也取决于 ParamType 的类型，有三种情况

  • ParamType 是一个指针或引用，但不是通用引用
  • ParamType 是一个通用引用
  • ParamType 既不是指针也不是引用

ParamType 是一个指针或引用，但不是通用引用​

这种情况的推导规则如下：

  • 如果 expr 的类型是一个引用，忽略引用部分
  • 然后 expr 的类型与 ParamType 进行模式匹配来决定 T

比如

template<typename T>
void f(T& param);               //param是一个引用

如果传入以下实参

int x=27;                       //x是int
const int cx=x;                 //cx是const int
const int& rx=x;                //rx是指向作为const int的x的引用

最后会推导出

f(x);                           //T是int，param的类型是int&
f(cx);                          //T是const int，param的类型是const int&
f(rx);                          //T是const int，param的类型是const int&

可以看到 cx 和 rx 都正确推导出了 const int，但是 rx 原本是一个 const int& 类型的。这里和第一点有关，即实参的引用性(reference-ness)在类型推导中会被忽略

ParamType 是一个通用引用​

模板通用引用的形式和右值引用的形式相同，推导规则如下：

  • 如果 expr 是左值， T 和 ParamType 都会被推导为左值引用
  • 如果 expr 是右值， 就使用上面正常的推导规则

比如

template<typename T>
void f(T&& param);              //param现在是一个通用引用类型
        
int x=27;                       //如之前一样
const int cx=x;                 //如之前一样
const int & rx=cx;              //如之前一样

f(x);                           //x是左值，所以T是int&，
                                //param类型也是int&

f(cx);                          //cx是左值，所以T是const int&，
                                //param类型也是const int&

f(rx);                          //rx是左值，所以T是const int&，
                                //param类型也是const int&

f(27);                          //27是右值，所以T是int，
                                //param类型就是int&&

ParamType 既不是指针也不是引用​

当 ParamType 既不是指针也不是引用的时候，通过传值的方式处理，相当于 param 是传入实参的一个拷贝，推导规则如下：

  • 如果 expr 的类型是一个引用，忽略引用
  • 如果忽略完引用之后， expr 是一个 const ，则忽略 const ；如果是一个 volatile ，则继续忽略 volatile。以此类推

因此

template<typename T>
void f(T param);                //以传值的方式处理param

int x=27;                       //如之前一样
const int cx=x;                 //如之前一样
const int & rx=cx;              //如之前一样

f(x);                           //T和param的类型都是int
f(cx);                          //T和param的类型都是int
f(rx);                          //T和param的类型都是int

即使 cx 和 rx 表示的是 const ，但这并不意味着 param 并不能修改。 param 是作为 cx 或 rx 的一个拷贝，是一个独立的对象。

数组实参​

C++中数组可以退化为指针，数组名可以作为数组第一项地址的指针，比如

const char name[] = "J. P. Briggs";     //name的类型是const char[13]

const char * ptrToName = name;          //数组退化为指针

需要注意的是，这里 name 的类型是 const char[13]，因为可以退化成指针，所以 const char* 可以由 name 初始化

当我们将一个数组传值给一个模板会推导成什么？

template<typename T>
void f(T param);                        //传值形参的模板

f(name);                                //T和param会推导成什么类型?

这里传 name 之后， T 会被推导成 const char*，因为规则将 int param[] 与 int *param 视为等价的。

那怎么样才能推导成真正的数组呢？虽然函数不能声明形参为真正的数组，但是可以接受指向数组的引用，我们将模板改为传引用

template<typename T>
void f(T& param);                       //传引用形参的模板

f(name);

这里 T 就被推导成 const char[13]了

函数实参​

与数组实参类似

void someFunc(int, double);         //someFunc是一个函数，
                                    //类型是void(int, double)

template<typename T>
void f1(T param);                   //传值给f1

template<typename T>
void f2(T & param);                 //传引用给f2

f1(someFunc);                       //param被推导为指向函数的指针，
                                    //类型是void(*)(int, double)
f2(someFunc);                       //param被推导为指向函数的引用，
                                    //类型是void(&)(int, double)

总结一下：

  • 在模板类型推导时，有引用的实参会被视为无引用，他们的引用会被忽略
  • 对于通用引用的推导，左值实参会被特殊对待
  • 对于传值类型推导，const和/或volatile实参会被认为是non-const的和non-volatile的
  • 在模板类型推导时，数组名或者函数名实参会退化为指针，除非它们被用于初始化引用
