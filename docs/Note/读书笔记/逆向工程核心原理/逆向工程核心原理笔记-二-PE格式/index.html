逆向工程核心原理笔记(二) | PE格式

PE文件格式​

PE文件由两个部分组成：PE头部分、PE体，这里以32位的notepad.exe进行笔记

PE_header​

IMAGE_DOS_HEADER​

typedef struct _IMAGE_DOS_HEADER
{
     WORD e_magic;         // 'MZ'
     WORD e_cblp;
     WORD e_cp;
     WORD e_crlc;
     WORD e_cparhdr;
     WORD e_minalloc;
     WORD e_maxalloc;
     WORD e_ss;
     WORD e_sp;
     WORD e_csum;
     WORD e_ip;
     WORD e_cs;
     WORD e_lfarlc;
     WORD e_ovno;
     WORD e_res[4];
     WORD e_oemid;
     WORD e_oeminfo;
     WORD e_res2[10];
     LONG e_lfanew;         // offset to NT_header
} IMAGE_DOS_HEADER, *PIMAGE_DOS_HEADER;

IMAGE_DOS_HEADER中有两个重要的成员：

  • e_magic：DOS签名（value=4D5A => 'MZ'）
  • e_lfanew：IMAGE_NT_HEADERS的便宜

IMAGE_DOS_HEADER下还有一段IMAGE_DOS_STUB，没有这一段也可以运行。这一段的意义就是输出This program cannot be run in DOS mode

IMAGE_NT_HEADERS​

typedef struct _IMAGE_NT_HEADERS
{
     DWORD Signature;
     IMAGE_FILE_HEADER FileHeader;
     IMAGE_OPTIONAL_HEADER OptionalHeader;
} IMAGE_NT_HEADERS, *PIMAGE_NT_HEADERS;

IMAGE_NT_HEADERS由三个成员构成：

  • Signature：签名（value=50450000 => 'PE00'）
  • FileHeader：文件头
  • OpitonalHeader：可选头

IMAGE_OPTIONAL_HEADER [32 | 64]​
typedef struct _IMAGE_DATA_DIRECTORY
{
     DWORD VirtualAddress;
     DWORD Size;
} IMAGE_DATA_DIRECTORY, *PIMAGE_DATA_DIRECTORY;

typedef struct _IMAGE_OPTIONAL_HEADER
{
     WORD Magic;
     BYTE MajorLinkerVersion;
     BYTE MinorLinkerVersion;
     DWORD SizeOfCode;
     DWORD SizeOfInitializedData;
     DWORD SizeOfUninitializedData;
     DWORD AddressOfEntryPoint;
     DWORD BaseOfCode;
     DWORD BaseOfData;
     DWORD ImageBase;
     DWORD SectionAlignment;
     DWORD FileAlignment;
     WORD MajorOperatingSystemVersion;
     WORD MinorOperatingSystemVersion;
     WORD MajorImageVersion;
     WORD MinorImageVersion;
     WORD MajorSubsystemVersion;
     WORD MinorSubsystemVersion;
     DWORD Win32VersionValue;
     DWORD SizeOfImage;
     DWORD SizeOfHeaders;
     DWORD CheckSum;
     WORD Subsystem;
     WORD DllCharacteristics;
     DWORD SizeOfStackReserve;
     DWORD SizeOfStackCommit;
     DWORD SizeOfHeapReserve;
     DWORD SizeOfHeapCommit;
     DWORD LoaderFlags;
     DWORD NumberOfRvaAndSizes;
     IMAGE_DATA_DIRECTORY DataDirectory[16];
} IMAGE_OPTIONAL_HEADER, *PIMAGE_OPTIONAL_HEADER;

IMAGE_OPTIONAL_HEADER中重要的成员如下：

  • Magic：32位和64位的标志位，10B代表32位，20B代表64位

  • AddressOfEntryPoint：指出程序最先执行的代码的起始地址
  • ImageBase：指出文件的优先载入地址

EXE、DLL文件装入用户内存的0-7FFFFFFF中，SYS文件被载入内核内存的80000000-FFFFFFFF中

  • SectionAlignment, FileAlignment：FileAlignment指定节区在磁盘文件中的最小单位，SectionAlignment指定节区在内存中的最小单位；两者可能相同可能不同，磁盘文件或内存的节区大小必定为FileAlignment或SectionAlignment的整数倍
  • SizeOfImage：指定PE Image在虚拟内存中所占的空间大小
  • SizeOfHeaders：指出整个PE头的大小
  • Subsystem：区分系统驱动文件和普通的可执行文件
值含义备注
1Driver文件系统驱动
2GUI文件窗口应用程序
3CUI文件控制台应用程序
  • NumberOfRvaAndSizes：指定IMAGE_OPTIONAL_HEADER最后一个成员DataDirectory数组大小
  • DataDirectory：由IMAGE_DATA_DIRECTORY定义的数组，数组每项都有被定义的值

这里面比较重要的是：EXPORT/IMPORT/RESOURCE、TLS Direction

IMAGE_SECTION_HEADER​

#define IMAGE_SIZEOF_SHORT_NAME

typedef struct _IMAGE_SECTION_HEADER
{
     BYTE Name[IMAGE_SIZEOF_SHORT_NAME];
     union {
        DWORD PhysicalAddress;
        DWORD VirtualSize;
     } Misc;
     DWORD VirtualAddress;
     DWORD SizeOfRawData;
     DWORD PointerToRawData;
     DWORD PointerToRelocations;
     DWORD PointerToLinenumbers;
     WORD NumberOfRelocations;
     WORD NumberOfLinenumbers;
     DWORD Characteristics;
} IMAGE_SECTION_HEADER, *PIMAGE_SECTION_HEADER;

IMAGE_SECTION_HEADER中比较重要的成员：

  • VirtualSize：内存中节区所占的大小
  • VirtualAddress：内存中节区的起始地址（RVA）
  • SizeOfRawData：磁盘文件中节区所占大小
  • PointerToRawData：磁盘文件中节区的起始地址
  • Characteristics：节区属性

PE文件内存地址与文件偏移的映射（RVA => RAW）

RAW - PointerToRawData = RVA - VirtualAddress

RAW = RVA - VirtualAddress + PointerToRawData

IMAGE_SECTION_DATA​

这里是每个节区的数据，节区头和节区之间、节区和节区之间都有NULL填充

IMAGE_IMPORT_DESCRIPTOR​

IMAGE_IMPORT_DESCRIPTOR中记录PE文件需要导入哪些库文件

typedef struct _IMAGE_IMPORT_DESCRIPTOR {
    union {
        DWORD Characteristics;
        DWORD OriginalFirstThunk;
    };
    DWORD TimeDateStamp;
    DWORD ForwarderChain;
    DWORD Name;
    DWORD FirstThunk;
} IMAGE_IMPORT_DESCRIPTOR;

typedef struct _IMAGE_IMPORT_BY_NAME {
    WORD Hint;
    BYTE Name[1];
} IMAGE_IMPORT_BY_NAME, *PIMAGE_IMPORT_BY_NAME;

IMAGE_IMPORT_DESCRIPTOR中重要的成员如下：

  • OriginalFirstThunk：INT的地址
  • Name：库名称字符串的地址
  • FirstThunk：IAT的地址

关于INT和IAT，以kernel32.dll为例

IMAGE_EXPORT_DESCRIPTOR​

EAT​

typedef struct _IMAGE_EXPORT_DIRECTORY {
    DWORD Characteristics;
    DWORD TimeDateStamp;
    WORD  MajorVersion;
    WORD  MinorVersion;
    DWORD Name;
    DWORD Base;
    DWORD NumberOfFunctions;
    DWORD NumberOfNames;
    DWORD AddressOfFunctions;
    
    DWORD AddressOfNames;
    DWORD AddressOfNameOrdinals;
} IMAGE_EXPORT_DIRECTORY, *PIMAGE_EXPORT_DIRECTORY;

IMAGE_EXPORT_DIRECTORY中几个重要的成员如下：

  • NumberOfFunctions：实际Export函数的个数
  • NumberOfNames：Export函数中具名的函数个数
  • AddressOfFunctions：Export函数地址数组（个数=NumberOfFunctions）
  • AddressOfNames：函数名称地址数组（个数=NumberOfNames）
  • AddressOfNameOrdinals：Ordinal地址数组（个数=NumberOfNames）

以kernel32.dll为例子

下面用GetProcAddress()的操作原理来理解EAT：

  1. 利用AddressOfNames转到函数名数组地址
  2. 用strcmp()比较函数名得到name_index
  3. 利用AddressOfNameOrdinals转到ordinal数组
  4. 在ordinal数组中通过name_index得到函数对应的ordinal值
  5. 利用AddressOfFunctions转到函数地址数组的地址
  6. 将ordinal值用作数组索引，获得函数的起始地址
